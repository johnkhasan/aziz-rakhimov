# Aziz Rahimov Blog

An Obsidian vault containing notes and writings from the Aziz Rahimov blog (@azizrakhimov_blog Telegram channel archive). All content is in Uzbek.

This repository **is** a ready-to-use Obsidian vault — the `.obsidian/` folder at the root holds the vault configuration, and every `.md` file is a note.

## How to import into Obsidian

### Option 1 — Clone with Git (recommended)

Keeps the vault in sync with the repository so you can `git pull` future updates.

1. Install [Obsidian](https://obsidian.md/download) if you don't already have it.
2. Clone the repo to a location of your choice:
   ```bash
   git clone https://github.com/<owner>/<repo>.git "Aziz Rahimov Blog"
   ```
3. Open Obsidian.
4. On the start screen, click **Open folder as vault** (or **Open another vault → Open folder as vault** if you already have a vault open).
5. Select the cloned `Aziz Rahimov Blog` folder.
6. Obsidian will load the vault using the configuration in `.obsidian/`.

To pull future updates:
```bash
cd "Aziz Rahimov Blog"
git pull
```

### Option 2 — Download the pre-built vault ZIP

Use this if you don't have Git or don't need updates. A ready-to-open vault archive is in `releases/aziz-rahimov-blog-v1.0.0.zip`.

1. Open `releases/aziz-rahimov-blog-v1.0.0.zip` on the GitHub repo page and click **Download raw file**.
2. Unzip the archive somewhere on your computer — it will extract a folder called `aziz-rahimov-blog-staging`.
3. Rename that folder to `Aziz Rahimov Blog` (optional).
4. In Obsidian, click **Open folder as vault** and select the unzipped folder.

### Option 3 — Download the whole repo as ZIP

1. On the GitHub repo page, click **Code → Download ZIP**.
2. Unzip, then open the folder as an Obsidian vault.

## Notes

- The vault's start page is **`Aziz Rahimov Blog - MOC.md`** (Map of Content) — open it first for an index of all notes.
- Wikilinks (`[[Note Name]]`) resolve automatically once the folder is opened as a vault.
- Local Obsidian state (`workspace.json`, `.trash/`) is gitignored so your personal layout won't conflict with the repo.
